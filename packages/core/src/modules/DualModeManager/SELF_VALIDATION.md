# DualModeManager 自我验证文档 v1.0

## 系统简介

联网/离线双模式智能切换管理器是一个确保AI小说创作引擎在任何网络环境下都能正常工作的核心模块。通过智能监测网络状态和自动切换工作模式，确保系统功能的连续性和可靠性。

## 功能清单

### ✅ 已实现的核心功能

1. **网络状态监测**
   - 实时检测网络连通性（支持多个fallback URL）
   - 测量网络延迟
   - 自动重连检测
   - 状态变化事件通知

2. **本地API服务器**
   - 支持用户配置API密钥（OpenAI、Anthropic、DeepSeek等）
   - 本地代理转发请求
   - 请求缓存（TTL可配置）
   - 速率限制保护
   - SSL/TLS支持（可选）

3. **智能模式切换**
   - 三种模式：online（纯在线）、offline（纯离线）、hybrid（智能切换）
   - 自动切换（网络恢复/丢失时）
   - 手动切换（用户强制指定）
   - 防抖处理（避免频繁切换）
   - 切换历史记录

4. **离线模式支持**
   - 基础向量搜索
   - 规则匹配审计
   - 一致性检查
   - 所有数据管理功能可用
   - 用户输入驱动的创作功能

5. **能力感知**
   - 实时能力状态查询
   - 按功能的能力可用性报告
   - 模式切换时的能力变化通知

### 🔄 待完善的功能

1. **高级故障转移**
   - 多API密钥自动轮换
   - API服务商故障时的自动切换
   - 降级策略（高质量→中等质量→基础模式）

2. **离线LLM支持**
   - Ollama集成
   - 本地模型缓存
   - 离线推理引擎

## 能力边界

### 明确能做到的

1. ✅ 实时监测网络连通性
2. ✅ 自动在网络恢复时切换到在线模式
3. ✅ 自动在网络丢失时切换到离线模式
4. ✅ 提供本地API代理服务
5. ✅ 在离线模式下提供基础功能
6. ✅ 缓存API响应以提高性能
7. ✅ 提供完整的模式切换历史
8. ✅ 支持多种API提供商

### 明确做不到的

1. ❌ 不提供离线LLM推理（需要额外配置Ollama等）
2. ❌ 不保证100%的网络可用性（取决于实际网络环境）
3. ❌ 不提供自动翻译服务
4. ❌ 不提供AI写作功能（在离线模式下需要用户手动输入）
5. ❌ 不处理语音输入或输出
